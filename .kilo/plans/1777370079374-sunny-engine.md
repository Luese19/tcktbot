# Plan to Add Welcome Message for Employee Private DMs

## Objective
Add a welcome message that is sent to registered employees when they send a private DM to the bot for the first time.

## Current State Analysis
- The bot uses python-telegram-bot library
- Main handler setup is in `main.py`
- Employees register via `/register_email` command and are stored in `data/employees/registrations.json`
- No current handler for private text messages
- Welcome messages exist only for bot joining groups, not for private DMs

## Implementation Plan

### 1. Extend Employee Service
- Add methods to track and check if an employee has been welcomed
- Add `welcomed` field to employee registry (default False)
- Methods: `mark_as_welcomed(user_id)`, `is_welcomed(user_id)`

### 2. Create Private DM Welcome Handler
- Create a new handler class `PrivateWelcomeHandler` in `handlers/` directory
- Handler filters: `filters.ChatType.PRIVATE & filters.TEXT`
- Logic:
  - Check if user is registered employee (`EmployeeService.is_registered(user_id)`)
  - If registered and not welcomed (`!EmployeeService.is_welcomed(user_id)`):
    - Send welcome message
    - Mark as welcomed (`EmployeeService.mark_as_welcomed(user_id)`)
- Welcome message content:
  ```
  Welcome to the Help Desk Bot!

  You can use these commands:
  - Create a new ticket: /start
  - View your tickets: /my_tickets  
  - Check ticket status: /ticket_status
  - Get help: /help
  - Register/update email: /register_email
  ```

### 3. Integrate Handler into Main Bot
- Add the handler to `main.py` in `_setup_handlers()`
- Use group priority 0 (high priority, before conversation handler)
- Import and add: `self.application.add_handler(PrivateWelcomeHandler.get_handler(), group=0)`

### 4. Testing
- Test with registered employee sending private message
- Verify message is sent only once
- Test with non-registered user (should not send welcome)
- Test commands still work in private

## Files to Modify
- `services/employee_service.py` - Add welcome tracking methods
- `handlers/private_welcome_handler.py` - New file for the handler
- `main.py` - Add handler registration

## Assumptions
- Welcome message sent only to registered employees
- Welcome sent only once per employee
- Message is sent on first private text message after registration
- Non-registered users do not receive welcome message</content>
